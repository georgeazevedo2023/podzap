## 5. Skills procedurais (`.claude/skills/`)

Quatro skills criadas pra automatizar fluxos repetidos. Todas com frontmatter `name` + `description` (com triggers).

### `podzap-test-webhook`
- Triggers: "testar webhook", "simular áudio", "mandar webhook fake"
- Gera body wsmart-shape, calcula HMAC com env, POST + verifica DB
- Variantes documentadas: text/audio/image/reaction, fromMe, .enc URL
- Códigos HTTP esperados + diagnóstico (401 = secret diff; 500 = HMAC env missing; 504 = container reiniciando)

### `podzap-deploy`
- Triggers: "deploy", "atualizar prod", "redeploy"
- Pipeline: garantir push → watch CI (`gh run watch`) → Portainer webhook → wait via `ScheduleWakeup` → probe `/login` + `/api/webhooks/uazapi`
- Documenta quando NÃO basta o webhook (env vars novas precisam "Update the stack" manual no Portainer)

### `podzap-migration`
- Triggers: "criar migration", "nova migration", "alterar tabela"
- Fluxo: next number → template SQL idempotente → `db-query.mjs` → `gen-types.mjs` → atualizar service layer + tests → commit padronizado
- Princípios: idempotente, comments em colunas novas, CHECK em vez de drop enum value (padrão 0011)

### `podzap-db`
- Triggers: "consulta DB", "select", "quantos", "rodar SQL"
- Wrapper pra `node --env-file=.env.local scripts/db-query.mjs --sql "..."`
- Padrões úteis: contagens, forensics (parser bug discovery), status tenant/instância, pipeline state
- Cuidados: limit, LGPD em conteúdo de mensagens, DELETE/UPDATE com confirmação
