## Screenshots-descritos

### Logo — antes / depois

- **antes**
  - 40×40 tile roxo (`--purple-600`) rotacionado −4° com 🎙 branco; shadow chunky 3×3.
  - stack lettering à direita: linha 1 "pod**ZAP**" 22px Archivo Black (`--font-brand`), "ZAP" em `--pink-500` flat (sem sombra).
  - linha 2 "zap → podcast" 9px caps, `letter-spacing: 0.2em`, `--text-dim`, `marginTop: 2`.

- **depois**
  - mesma tile (intacta — é o icon do protótipo).
  - "ZAP" agora com `text-shadow: 2px 2px 0 var(--stroke)` — em dark mode o stroke é puro `#000`, então o offset vira o degradê chunky-comic que o protótipo tem no thumbnail (o "podZAP" de 72px na splash).
  - gap aumentado pra 4px — bate opticamente com o lettering da splash.

### Plan card sticker — antes / depois

- **antes**: "🔥 plano hype" renderizava **verde lime** (`#C6FF3C`) no canto do card — porque `.sticker.pink` não existe na Tailwind v4 layer, só o legacy `.sticker.pink` do `tokens.css` que nunca foi portado.
- **depois**: rosa shock (`#FF3DA5`) com texto branco, como previsto no mockup.

### Button variants — antes / depois

- **antes**: "upgradar o bagulho" no plan card era lime (caiu pro default). Todos os CTAs `variant="purple"` / `variant="pink"` / `variant="ghost"` no app idem.
- **depois**: lê roxo `--purple-600` com contrast branco, igual ao protótipo. (O mesmo fix se propaga por todos os `<Button variant=…>` do app — 24+ call sites.)

## Itens pendentes

1. **Landing (`/`) e login (`/login`)** são telas custom fora do protótipo (ver `CLAUDE.md §18`). Não foram tocadas.
2. **`TopBar`** passou o sanity-check — estrutura (eyebrow → h1 38px Bricolage → subtitle) bate com `podZAP/shell.jsx`. Single concern: o `fgMap` do mockup (`{ purple: '#fff', lime: '--ink-900', … }`) não existe no TopBar.tsx atual porque o título sempre usa `var(--text)`. Isso funciona no layout porque o TopBar renderiza sobre `--surface`, não sobre o accent — é o blob que recebe o accent. Mantido como está.
3. **`AdminSidebar`** usa `var(--radius-md)` (token Tailwind `@theme`) em vez de `var(--r-md)` (token `:root`). Ambos resolvem, mas é inconsistente com o `Sidebar.tsx` do tenant-app. Não alterei pra não misturar fix de logo com refactor de tokens — registrar como débito leve.
4. **SVG fidelity do thumbnail inicial** (o big splash `podZAP` 72px sobre fundo roxo com `stroke="#C6FF3C"`) só aparece no `podZAP _standalone_.html` como pré-loader SVG inline. Não é usado em lugar nenhum do app. Se o usuário quiser um "splash/loading" idêntico, vira uma feature nova (pós-MVP).
5. **Primitive audit completo** (Card, Modal, PodCover, Waveform, PlayerWave, MicMascot, RadioPill, Select, StatCard) não foi feito diff-por-diff — só as primitives que o usuário explicitamente chamou (logo + sticker/button via fallthrough). Recomendo agendar uma passada completa se surgir mais queixa de paridade.
